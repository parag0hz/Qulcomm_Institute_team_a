# CFA 실험 보고서 — CSV(테이블) 및 딥러닝(포인트클라우드) 학습 결과

**작성일** 2026-07-16 · **환경** RTX 5080 16GB, torch 2.13.0+cu130 (conda `qi`, py3.12) / AutoGluon 1.5.0 (conda `automl`, py3.11) · **실험 추적** [wandb `cfa` 프로젝트](https://wandb.ai/limcon00-chosun-university/CFA)

모든 수치는 별도 표기 없으면 **공식 test split**, seed 0 기준. 재현 명령은 §7.

## 1. 평가지표

| 지표 | 정의 | 답하는 질문 | 함정 |
|---|---|---|---|
| **R²** | 1 − SS_res/SS_tot (부분집합 자신의 평균이 기준선) | 평균 찍기보다 나은가 | 차종 간 비교 불가 — Estate σ(0.019)가 Fastback(0.038)의 절반이라 같은 절대오차도 나쁘게 보임 |
| **MAE (drag counts)** | mean\|ŷ−y\|×1000 (1 count = 0.001 Cd) | 물리적으로 얼마나 틀리나 | 차종 간 비교 가능, 공력 실무 단위. 문헌 수용 기준 MAE<5 counts |
| **쌍대 순위 정확도** | 무작위 두 설계 중 저항 낮은 쪽 적중률 | 제품이 실제로 하는 일 | 50% = 동전 던지기. **헤드라인 지표** |
| MAE % / Spearman ρ | 보조 | 상대오차 / 순위 상관 | |

원칙: 클린 전역 수치 하나만 보고하지 않는다. **차종별 분해 필수** (Fastback이 68%라 전역 평균이 실패를 감춤).

## 2. 트랙 A — CSV(테이블 파라미터) 학습

### 입력 / 출력

- **데이터**: `data/DrivAerNet_ParametricData.csv` — **4,165 설계 × 32열** (DrivAerNet++ 신규 파라메트릭 세대만; 구 fastback 3,966대는 50-파라미터 세대라 이 표에 없음)
- **입력 피처**: ① `geoparams` = 기하 설계 파라미터 **23개** (차체 길이/폭/루프높이, 램프·디퓨저·트렁크리드 각도 등) ② `paper` = 저자 코드 그대로 **29개** (23 + 양력계수 Cl 계열 6열 — ⚠ CFD 산출물로 설계 시점에 없는 **누수성 피처**)
- **출력**: Average Cd (스칼라 회귀)

### 사용 모델 · 학습 config

| 항목 | 값 |
|---|---|
| 모델 5종 | AutoGluon 1.5.0 (fit 무제한) / XGBoost 3.1.3 / LightGBM 4.6.0 / RandomForest / GradientBoosting (기본 하이퍼파라미터, random_state=42) |
| 분할 | 80/20 고정 (seed 42) — 논문 §5.1.2 프로토콜 |
| 학습분율 | 20/40/60/80/95(=100)% × **재추출 20회** 평균 ± 95% CI |
| 데이터셋 2종 | Fastback 단독(1,376) / 통합 F+N+E(4,165) |

### 결과 (100% 학습, test R²)

| 모델 | 통합 (geoparams) | 통합 (paper: Cl 포함) | Fastback (geoparams) |
|---|---:|---:|---:|
| AutoGluon | **0.549 ±0.005** | 0.805 ⚠ | **0.722 ±0.004** |
| LightGBM | 0.537 ±0.004 | 0.782 ⚠ | 0.633 ±0.004 |
| XGBoost | 0.513 ±0.006 | 0.753 ⚠ | 0.606 ±0.006 |
| GradientBoosting | 0.511 ±0.001 | 0.755 ⚠ | 0.699 ±0.003 |
| RandomForest | 0.467 ±0.004 | 0.729 ⚠ | 0.653 ±0.003 |

데이터 스케일링 (통합, geoparams): AutoGluon 0.458→0.549, LightGBM 0.376→0.537, XGBoost 0.305→0.513 (20%→100%).

### 논문(§5.1.2, Fig. 5)과 비교

| 항목 | 논문 | 우리 | 판정 |
|---|---|---|---|
| 통합 100% (AutoGluon/LGBM/XGB) | ~0.58 / ~0.60 / ~0.55 | 0.549 / 0.537 / 0.513 | ✅ 재현 (0.03~0.06 이내) |
| "XGBoost 640→3,200개에서 0.35→0.55" | 본문 명시 | 0.305→0.513 | ✅ 재현 |
| 모델 순위 (Fastback: AG>GB>RF>LGBM>XGB) | Fig.5 | 동일 순위 | ✅ 재현 |
| Fastback 100% AutoGluon | ~0.83 | 0.722 (무제한 fit·20회에도 동일) | ❌ 미재현 |
| **발견** | — | 저자 공개 코드는 Cl 피처를 입력에 포함 → 통합 R² +0.25 부풀림. 단일 피처 구성으로 두 패널 동시 재현 불가 | ⚠ 재현성 한계 |

**결론**: 전 차종 통합에서 테이블 파라미터의 천장 ≈ **R² 0.55** — 형상 기반 모델의 필요성을 뒷받침.

## 3. 트랙 B — 딥러닝(포인트클라우드) 학습

### 입력 / 출력

- **데이터**: DrivAerNet++ 포인트클라우드 미러 **7,713대 × 100k점** → GPU FPS 캐시 `data/fps2048.npz`
- **입력**: FPS-2048(또는 1024) 점 `(N, 3)` float32, **미터 스케일 유지**(불변식: 절대 높이 r=+0.83이 최강 신호), train 평균으로 상수 평행이동만
- **출력**: Cd 스칼라 (타깃은 train 통계로 표준화 후 회귀, 예측 시 역변환)
- **분할**: 공식 5,398 / 1,157 / 1,158 (재분할 금지, 모델 선택은 val만)

### 학습 config

| | 우리 레시피 (train_r2.py) | 저자 레시피 (RegDGCNN 재현용) |
|---|---|---|
| 옵티마이저 | AdamW lr 1e-3, wd 1e-4 | Adam lr 1e-3, wd 1e-4 |
| 손실 | SmoothL1(β=1), 표준화 타깃 | MSE, raw Cd |
| 스케줄러 | Cosine 120ep (eta_min 1e-5) | ReduceLROnPlateau(p20, ×0.1), 100ep |
| 배치 | 32 (공정 매트릭스는 8) | 8 |
| 조기종료/선택 | patience 30, best val R² | best val MSE |
| 증강 | 없음 | 없음 (저자 원본은 jitter 적용 — 재현에서 제외, §5 참고) |

### 결과 ① 베이스라인 사다리 (2048점, bs32)

| 모델 | params | R² | MAE (counts) | 순위 정확도 |
|---|---:|---:|---:|---:|
| 차종 평균만 | 0 | 0.226 | — | — |
| R0: 로버스트 치수 6 → 선형+2차 | 13 | 0.814 | 12.5 | 85.6% |
| R1: 치수 6 → MLP | 0.14M | 0.861 | — | — |
| 복셀 1-NN 검색 (학습 0) | 0 | 0.865 | — | — |
| DGCNN (1024점) | 1.80M | 0.951 | — | — |
| **PointNet 2048** | **0.81M** | **0.968** | **5.1** | **94.6%** |

PointNet 차종별: Fastback 0.974 (4.7 counts, 95.4%) / Estate 0.839 (5.5, 87.3%) / Notchback 0.802 (6.6, 86.2%). 시드 재현성: seed 0/1 전역 R² 동일(0.968), 차종별 ±0.03.

### 결과 ② 공정 비교 매트릭스 (완전 동일 조건: 1024점·bs8·seed0·우리 레시피)

- RegDGCNN 격차 분해: 레시피 교체 +0.007 (0.887→0.894), 아키텍처 교체 +0.056 (0.894→0.950) → **격차의 주범은 아키텍처**.
- Triplane-lite는 TripNet(0.957)의 대리가 아님 — 표현 아이디어만으로는 부족함을 확인한 하한선.

## 4. 논문(DrivAerNet++, NeurIPS 2024)과 종합 비교

| 모델 | 논문 Table 4 | 우리 측정 | 비고 |
|---|---:|---:|---|
| PointNet | 0.643 | **0.968** | 같은 아키텍처 계열, 파이프라인 차이 |
| RegDGCNN | 0.641 | 0.867 (저자 정규화·레시피 재현) / 0.894 (우리 레시피) | 저자 구현 그대로, 파라미터 3,164,257 일치 |
| GCNN | 0.596 | — | 메시 필요 (범위 밖) |
| TripNet (후속 문헌 SOTA) | 0.957 | — | **코드 비공개 → 문헌치 인용만** |

**격차의 원인 (코드 레벨로 규명)**: ① 논문 파이프라인은 클라우드별·축별 min-max 정규화로 절대 스케일·종횡비 소거 ② 저자 데이터셋 클래스는 val/test에도 1cm jitter 증강 적용(`apply_augmentations` 기본값 True) ③ 메시 정점 무작위 추출 + 제로패딩 ④ test 로더 drop_last로 일부 표본 누락. → 저자 코드·조건 그대로도 우리 데이터에서 0.867이 나오므로, 논문 수치는 아키텍처 한계가 아니라 실행상 문제로 판단.

**주장 가능/불가**: "공식 벤치마크 대비 +0.32, 문헌 SOTA와 동급 수치(0.81M, 학습 4분)" 가능. "SOTA 경신" 단정은 불가 — 미러 데이터(7,713 vs 8,000), 시드 2개, 그리고 공식 split R² 자체가 일반화를 과대평가함(1-NN만으로 0.865, 계열 홀드아웃 0.75~0.80)을 우리 스스로 실측.

## 5. 한계

1. 트랙 A(4,165대·80/20)와 트랙 B(7,713대·공식 3-way)는 **표본·분할이 달라 R² 직접 비교 불가** — "테이블 vs 형상"은 경향으로만 인용.
2. 딥러닝 결과 대부분 시드 1~2개 (차종별 시드 변동 ±0.03 실측 — 그 이하의 차이는 해석 금지).
3. 열화(폰스캔) 조건 딥모델 성능 미측정 — 로드맵 R3.
4. RegDGCNN 재현은 1024점(논문 5,000) — VRAM 제약, 점 수 민감도는 낮음(PointNet 1024↔2048 = +0.003).
