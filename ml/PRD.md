# PRD — CFA (Car Fluid Analyzer)

**버전** 0.1 (초안) · **작성일** 2026-07-14 · **기반 자료** CFA_HackathonFinal_v9.pdf, RESULTS.md, DATA_SUMMARY.md

---

## 1. 한 줄 정의

> 차량 3D 설계 파일(.stl/.obj)을 업로드하면 **수 초 안에 항력계수(Cd)를 예측**하고, 목표치 초과 여부를 알려주는 웹 기반 AI 도구. CFD 시뮬레이션의 대체재가 아니라, **설계 초기 단계의 빠른 사전 필터**.

## 2. 문제 정의

- 공기저항(항력)은 연비/전비를 좌우하는 핵심 요인이며, 항력 개선만으로 연료 절감 여지가 크다.
- 항력을 측정하는 표준 방법인 CFD는 **느리고 비싸다**: 고충실도 1개 설계에 수일~수주, 수퍼컴퓨터급 연산, 전문 엔지니어 팀 필요. (참고: DrivAerNet++ 데이터셋 구축에만 2,880 CPU 코어 / 39 TB.)
- 결과적으로 설계 초기 단계에서 **여러 후보안을 빠르게 비교하는 반복 작업이 불가능**하고, CFD 전문가가 아니면 접근 자체가 막혀 있다.

## 3. 목표 사용자

| 우선순위 | 사용자 | 니즈 |
|---|---|---|
| P0 | **자동차 디자이너 / 설계자** | CFD 지식 없이, 설계안의 공력 성능을 초기 단계에서 즉시 확인 |
| P1 | 제품 기획자 | 후보 설계군의 공력 비교로 의사결정 지원 |
| P2 | CFD 엔지니어 | 정식 CFD에 올릴 후보를 좁히는 사전 스크리닝 |

## 4. 핵심 기능 요구사항

| ID | 기능 | 설명 | 우선순위 |
|---|---|---|---|
| F1 | 3D 파일 업로드 | .stl/.obj 업로드 → 포인트클라우드 변환(샘플링) → 전처리 | **P0** |
| F2 | Cd 예측 | 업로드 후 **수 초 내** Cd 값 반환 (PointNet 서로게이트 모델) | **P0** |
| F3 | 목표치 판정 | 사용자가 설정한 목표 Cd 대비 초과/충족 상태 표시 | **P0** |
| F4 | 후보 비교(랭킹) | 여러 설계안을 올려 저항 낮은 순으로 정렬 — 모델이 가장 잘하는 일(순위 정확도 94.6%) | **P1** |
| F5 | 파라미터 기반 예측 | 3D 파일 없이 26개 설계 파라미터 입력 → LightGBM/XGBoost로 즉시 예측 (경량 트랙) | **P1** |
| F6 | OOD 경고 | 학습 분포(세단 계열) 밖 입력(SUV/트럭 등) 감지 시 신뢰도 경고 표시 | **P1** |
| F7 | 설계 수정 가이드 | 목표 초과 시 개선 방향 제안 (예: 후미 형상 수정) | P2 (스트레치) |

## 5. 비기능 요구사항

- **응답 시간**: 업로드 → 결과까지 종단 간 10초 이내 (전처리 포함; 추론 자체는 1초 미만)
- **모델 경량성**: PointNet 0.81M 파라미터 — 단일 GPU는 물론 CPU 서빙도 가능한 크기
- **플랫폼**: 웹 인터페이스 (설치 불필요, 브라우저 접근)
- **입력 견고성**: 단위 불명 STL(mm/m 혼용), 비수밀(non-watertight) 메시에 대한 방어적 처리 — 모델이 미터 절대 스케일에 의존하므로 단위 추정/검증 필수

## 6. 성공 지표

**제품 헤드라인 지표는 쌍대 순위 정확도** — "두 설계 중 저항 낮은 쪽을 맞히는가"가 이 제품이 실제로 하는 일이다 (50% = 동전 던지기).

| 단계 | 목표 | 달성 현황 (공식 test 1,158대) |
|---|---|---|
| MVP | 학습 파이프라인 완성 | ✅ 완료 |
| 1차 | R² ≥ 0.50 | ✅ 치수 회귀만으로 0.814 |
| 2차 | R² ≥ 0.60 | ✅ **PointNet 0.968** |
| 오차 | MAE ≤ 0.01 | ✅ **0.0051** (5.1 drag counts) |
| 최종 | STL → PointNet → Cd 웹 데모 | 🔲 진행 중 |

내부 품질 기준 (전역 평균이 실패를 감추므로 **차종별 분해 필수**):

- 차종별 MAE ≤ 7 drag counts (현재 Fastback 4.7 / Estate 5.5 / Notchback 6.6 ✅)
- 차종별 순위 정확도 ≥ 85% (현재 95.4 / 87.3 / 86.2% ✅)
- 근거: 치수 기반 모델은 Estate에서 순위 정확도 51.9%(동전 던지기) — 포인트클라우드 모델이 87.3%로 끌어올린 것이 핵심 차별점

## 7. 범위 밖 (Out of Scope)

- **절대 Cd 인증/측정** — CFA는 상대 비교·사전 필터 도구다. 최종 검증은 고충실도 CFD/풍동의 몫 (덱 슬라이드 9의 포지셔닝과 일치)
- 유동장(velocity/pressure field) 시각화 — Cd 단일 스칼라만 예측
- 세단 계열 밖 차종(SUV·트럭·상용차)의 신뢰성 있는 예측 — 학습 데이터가 DrivAer 세단 변형뿐
- 휴대폰 LiDAR 스캔 입력 — 향후 확장 후보 (부분 스캔·노이즈 열화 대응 연구 필요)

## 8. 기술 접근 요약

```
[트랙 1: ML]  26개 설계 파라미터(CSV) → LightGBM / XGBoost → Cd
[트랙 2: DL]  STL/OBJ → 포인트클라우드 샘플링 → PointNet(0.81M) → Cd
                (mesh 직접 입력 RegDGCNN과 비교 평가 → PointNet 채택)
```

- **데이터**: DrivAerNet++ (Elrefaie et al. 2024) — 세단 변형 7,713대, 차량당 10만 점, CFD 계산 Cd 라벨. 공식 split 5,398 / 1,157 / 1,158 사용 (재분할 금지)
- **설계 원칙**: ① 미터 스케일 유지 (절대 높이가 최강 신호, r=+0.83) ② 평가 시 차종별 분해 ③ 순위 정확도를 헤드라인 지표로

## 9. 리스크와 대응

| 리스크 | 영향 | 대응 |
|---|---|---|
| STL 단위 모호성 (mm/m) | 스케일 의존 모델이 완전히 틀린 값 출력 | 업로드 시 bbox 크기로 단위 추정 + 사용자 확인 UI |
| OOD 입력 (비세단) | 자신 있는 오답 | F6 OOD 플래그, 신뢰 구간 표시 |
| 실차 스캔·실측 Cd 검증 부재 | 시뮬레이션 데이터 내 성능만 입증됨 | 제조사 공표 Cd가 있는 실차 5~10대 스캔을 검증셋으로 확보 (학습 사용 금지) |
| 사용자 메시 품질 (구멍, 노이즈) | 전처리 실패 또는 성능 저하 | 포인트 샘플링 기반 파이프라인은 수밀성 불필요 — 열화 조건 학습(R3)으로 보강 |
