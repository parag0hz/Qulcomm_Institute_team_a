# Paragon Web Service

`Paragon` is a vehicle-aerodynamics design workspace built as a separate product layer on top of DrivAerNet. The original research and training code is not modified by this web application.

The web stack is now:

- FastAPI and Uvicorn for the eight JSON/file-upload API routes, validation, provider fallback, and production static serving.
- React, Vite, and TypeScript for the designer workspace.
- Zustand for the current design, baseline, history, locks, variants, and recommendation preview.
- Three.js, wrapped by a React component, for the QEM GLB viewer, approximate morphing, procedural wheels, and dimension overlays.

## Prerequisites

- Python 3.10–3.12 (Python 3.12 recommended for the pinned scikit-learn runtime)
- Node.js 20.19+ (20.x) or 22.12+ and npm
- The repository's tabular CSV and generated local model artifact for Local RandomForest prediction
- Optional: Google Cloud SDK for Vertex AI and an OpenAI API key for LLM-backed Copilot responses

Create an isolated Python environment, then install the web runtime and frontend dependencies from the repository root:

```bash
python3.12 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -r backend/requirements-web.txt
npm install
```

Keep this virtual environment active for the FastAPI, model-training, and Python test commands below. This prevents macOS from mixing a Homebrew Python with another framework Python.

The web requirements are intentionally separate from model-training and mesh-processing dependencies.

## Development

Run FastAPI and Vite in separate terminals.

Terminal 1 — API and model services:

```bash
uvicorn backend.cfa_service.app:app \
  --host 127.0.0.1 \
  --port 8001 \
  --reload
```

Terminal 2 — React development server:

```bash
npm run dev:web
```

Open `http://127.0.0.1:5173`. Vite proxies `/api` and `/static/models` to FastAPI at `http://127.0.0.1:8001`, so browser requests use the same paths as the production build.

FastAPI's development documentation is available at:

- `http://127.0.0.1:8001/docs`
- `http://127.0.0.1:8001/redoc`
- `http://127.0.0.1:8001/openapi.json`

## Production-Style Local Run

Build the React app, then start only FastAPI:

```bash
npm run build:web

uvicorn backend.cfa_service.app:app \
  --host 127.0.0.1 \
  --port 8001
```

Open `http://127.0.0.1:8001`. FastAPI serves `frontend/dist` and the GLB assets under `/static/models` from the same origin. If the React build is missing, the root page returns an instruction to run `npm run build:web`; API routes and API documentation remain separate.

## API Compatibility

The migration preserves the existing endpoints:

```text
GET  /api/status
GET  /api/parameters
POST /api/predict
POST /api/predict/parameters
POST /api/analyze/parameters
POST /api/optimize/parameters
POST /api/copilot
POST /api/providers/vertex/test
```

Pydantic validates the 23 numeric parameters, `CarRear`, `Wheels`, optimization locks and target, Copilot content, and Vertex test requests. Invalid fields return FastAPI `422 detail[]` entries. STL uploads accept only `.stl` files and are limited to 32 MB. Service failures use the shared `{"error": {"code": ..., "message": ...}}` response shape.

## Current Designer Workflow

- Selects body architecture and wheel treatment.
- Exposes all 23 numeric design parameters as bounded controls.
- Displays an optimized GLB built from the real `F_D_WM_WW_3532` DrivAer STL.
- Uses QEM to preserve a closed surface and exports named body plus four-wheel nodes.
- Changes the visible wheel treatment immediately: dataset Open detailed or procedural Open/Closed smooth.
- Morphs the reference mesh approximately as supported sliders move, always recalculating from the original vertices.
- Shows live length, width, roof-height, angle, and local-feature dimension annotations in the 3D viewport.
- Provides orbit, zoom, pan, shadows, materials, and preset engineering views.
- Predicts Cd with a trained local tabular model and compares it with DrivAerNet.
- Shows baseline delta, dataset percentile, training-domain distance, uncertainty, and explicit warnings.
- Calculates local ±5% sensitivity and links the top drivers back to sliders and 3D annotations.
- Saves up to five browser-local variants with thumbnails, comparison, undo/redo, and presets.
- Migrates a saved `paragon.workspace.v1` browser workspace into the current v2 format.
- Searches bounded, unlocked parameters for three target-Cd improvement directions.
- Exports a complete JSON record and an HTML report that can be printed or saved as PDF.
- Supports a separate STL upload path with a clearly marked geometry fallback.

The central mesh uses smooth coordinate-region masks for a fast designer-preview morph. It is not CFD-ready or CAD-accurate geometry: small unsegmented parts such as handles and pillars are annotated instead of being forced into unreliable local deformation. The Open detailed wheels come from the dataset STL; Open smooth and Closed smooth are clearly marked procedural visualizations fitted to the measured wheel bounds. Body architecture changes still require dedicated meshes. Precise production morphing still requires segmented Blender shape keys, CAD, or ANSA. The STL upload fallback is not a trained CFD surrogate and remains lower confidence than the parametric prediction.

## Prepare Reference Geometry

Install the QEM dependency:

```bash
python -m pip install -r backend/models/requirements-mesh.txt
```

Convert the local DrivAer STL into the topology-checked multi-part GLB:

```bash
python backend/models/prepare_reference_mesh.py \
  --input stl/3DMeshesSTL/F_D_WM_WW_8/F_D_WM_WW_3532.stl \
  --output backend/cfa_service/static/models/drivaer_reference.glb \
  --body-faces 80000 \
  --wheel-faces 8000
```

`prepare_reference_mesh.py` uses `fast-simplification` QEM. It separates the watertight source into one body and four wheels, simplifies every component independently, checks open and non-manifold edges, rebuilds normals, and atomically writes a multi-node GLB. The default output is 112,000 faces with zero open boundary edges. The 5+ GB local `stl/` source folder is intentionally ignored by Git.

To use another reference STL, provide a different `--input` path and rebuild the web asset. The defaults of 80,000 body faces and 8,000 faces per wheel balance topology, detail, and browser performance for the current reference STL.

## Train the Parametric Baseline

Install training dependencies and build the local model once:

```bash
python -m pip install -r backend/requirements-train.txt
python backend/models/train_parametric_baseline.py
```

The script uses `ParametricModels/DrivAerNet_ParametricData.csv`, derives body and wheel categories from `Experiment`, and trains a scikit-learn RandomForest by default so the artifact can always be loaded by the web runtime. It writes:

```text
backend/artifacts/cfa_parametric_baseline.pkl
```

The current local model schema is 23 numeric design parameters plus body and wheel categories. LightGBM and XGBoost are explicit alternatives via `--model lightgbm` or `--model xgboost`; when deploying one of those artifacts, install the selected estimator package in the serving environment as well. The web service continues to expose the actual model used for each prediction.

## Optional Vertex AI Provider

Local RandomForest remains the default because it supports the full 23 numeric plus two categorical feature schema. The current AutoML CSV contains only the 23 numeric inputs. To route numeric predictions to a deployed Vertex AI endpoint, copy the environment template, fill in the actual values locally, and authenticate with Application Default Credentials:

```bash
cp backend/.env.example backend/.env
gcloud auth application-default login
```

The relevant server-only values are:

```bash
PARAGON_PROVIDER=vertex
VERTEX_PROJECT_ID="your-gcp-project"
VERTEX_LOCATION="us-central1"
VERTEX_ENDPOINT_ID="your-endpoint-id"
```

Restart Uvicorn after changing `backend/.env`. A Vertex timeout, authentication error, or response-schema mismatch falls back to the local 25-feature model. The actual provider and fallback reason are shown in Confidence and included in exported reports.

The Confidence tab includes `Test Vertex connection`. It sends the current 23 numeric inputs directly to the configured endpoint and reports the returned Cd and latency. The server uses Google Application Default Credentials; `gcloud auth application-default login` is the local-development authentication path. Keep provider credentials and project settings on the FastAPI server, never in `VITE_` frontend variables.

## Optional Design Copilot

The Copilot tab is grounded in Paragon's local prediction, domain analysis, uncertainty, sensitivity, and bounded target search. It does not ask an LLM to invent a Cd. Without an API key it returns a deterministic evidence summary. To enable natural-language reasoning, set these only in `backend/.env`:

```bash
OPENAI_API_KEY="your-server-side-key"
OPENAI_MODEL="gpt-5-mini"
```

The key is never sent to the browser. The server calls the OpenAI Responses API and falls back to the grounded local explainer on network, authentication, or model errors. Restart Uvicorn after changing the environment file.

## Tests and Checks

Run frontend type checking, component tests, and the production build:

```bash
npm run typecheck
npm run test:web
npm run build:web
```

Run the FastAPI contract and core service tests, then compile-check the Python web package:

```bash
python -m unittest discover -s backend/tests -p 'test_*.py'
python -m compileall -q backend/cfa_service backend/models backend/tests
```

The Python API tests cover the eight routes, OpenAPI documents, field-level validation, the 32 MB STL boundary, common error responses, Vertex success/timeout/schema failures, and the 60-second fallback circuit breaker. The model-backed integration tests run when `backend/artifacts/cfa_parametric_baseline.pkl` exists; API contracts remain testable in a clean clone with deterministic fixtures. Browser smoke testing should use the production-style local run when checking the real GLB, morphing, wheel selection, camera views, dimensions, variants, optimizer, and Copilot together.
